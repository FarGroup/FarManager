OpenInfo

главная | структуры
Структура OpenInfo используется для возвращения Far Manager'ом информации о плагине в функцию OpenW. 
struct OpenInfo
{
  size_t StructSize;
  enum OPENFROM OpenFrom;
  const GUID *Guid;
  INT_PTR Data;
};

Элементы

StructSize
Это поле содержит размер структуры OpenInfo.
OpenFrom
Идентификатор, определяющий, откуда был вызван плагин. Может принимать одно из следующих значений (перечисление OPENFROM):
КонстантаОписание
OPEN_LEFTDISKMENU Открыт из левого меню "Сменить диск" (Alt-F1).
OPEN_PLUGINSMENU Открыт из меню "Команды внешних модулей" (F11).
OPEN_FINDLIST Открыт из диалога "Поиск файла". Этот идентификатор плагин получит только в том случае, если он экспортирует функцию SetFindListW. Последующий вызов функции SetFindListW произойдёт только в том случае, если функция OpenPanelW вернёт значение отличное от INVALID_HANDLE_VALUE.
OPEN_SHORTCUT Открыт через ссылку на папку в меню "Команды" - "Ссылки на папки".
OPEN_COMMANDLINE Был открыт из командной строки. Этот параметр может использоваться, только если плагин определил вызывающий префикс в функции GetPluginInfoW и этот префикс, с двоеточием после него, был указан в командной строке.
OPEN_EDITOR Открыт из редактора.
OPEN_VIEWER Открыт из встроенной программы просмотра.
OPEN_FILEPANEL Открыт из панелей.
OPEN_DIALOG Открыт из диалога.
OPEN_ANALYSE Открыт после анализа содержимого файла (например, архива)
OPEN_RIGHTDISKMENU Открыт из правого меню "Сменить диск" (Alt-F2).
OPEN_FROMMACRO Открыт из макрокоманды функцией callplugin; Data является числом.
OPEN_FROMMACROSTRING Открыт из макрокоманды. Item содержит указатель на строку. Этот флаг всегда идет с флагом OPEN_FROMMACRO, уточняет тип значения Data.
Guid
GUID выбранного элемента в списке пунктов, экспортированном данным плагином в меню "Диск", "Команды внешних модулей".
Data
Интерпретация параметра зависит от OpenFrom:
  • Если в параметре OpenFrom выставлен флаг OPEN_FROMMACRO, то значения в младшем слове содержат не идентификатор из перечисления OPENPLUGIN_OPENFROM, а номер текущей макрообласти, определенной в перечислении FARMACROAREA.
  • Для параметров OPEN_DISKMENU, OPEN_PLUGINSMENU, OPEN_EDITOR и OPEN_VIEWER Data - это номер выбранного пункта в меню из зарегистрированных плагином пунктов. Если плагин экспортирует только один элемент, этот параметр всегда равен нулю.
  • Для OPEN_FINDLIST Data всегда 0.
  • Для OPEN_SHORTCUT Data содержит адрес строки, которая была передана в элемент ShortcutData структуры OpenPlanelInfo в момент сохранения горячей клавиши. Плагин может использовать это поле для сохранения дополнительной информации о текущем состоянии. Не обязательно сохранять в нём информацию о текущей директории, так как этим занимается сам FAR.
  • Для OPEN_DIALOG Data содержит адрес структуры OpenDlgPluginData.
  • Для OPEN_COMMANDLINE Data содержит адрес введённого пользователем в командной строке выражения. Если плагин не установил флаг PF_FULLCMDLINE, то командный префикс плагина в командную строку не включается. При установленном флаге PF_FULLCMDLINE плагину передаётся полная строка (с командным префиксом).
    К примеру, если плагин устанавливает префикс ftp и пользователь вводит ftp://ftp.abc.com, Data будет указывать на строку: "//ftp.abc.com" при НЕустановленном флаге PF_FULLCMDLINE и "ftp://ftp.abc.com" если флаг PF_FULLCMDLINE установили.

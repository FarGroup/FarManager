_FAR_USE_WIN32_FIND_DATA

main | types and definitions

The macro _FAR_USE_WIN32_FIND_DATA controls whether the PluginPanelItem structure uses the WIN32_FIND_DATA structure defined in the Windows API headers or the the FAR_FIND_DATA that has the same layout but is defined in plugin.hpp. As some compilers (for instance, Borland C++ 5.5) force 8-byte alignment for structures defined in the Windows API headers, it is not possible to achieve the correct 2-byte alignment for the entire PluginPanelItem structure when the WIN32_FIND_DATA structure is used.

If the _FAR_USE_WIN32_FIND_DATA macro is defined, the PluginPanelItem structure will be compatible with FAR 1.65 and FAR 1.70 headers. But correct compilation will not be possible with Borland C++ 5.5 without modifying Windows API header files.

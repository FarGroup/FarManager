_FAR_USE_WIN32_FIND_DATA

main | types and definitions

The macro _FAR_USE_WIN32_FIND_DATA controls whether the PluginPanelItem structure uses the WIN32_FIND_DATA structure defined in the Windows API headers or the the FAR_FIND_DATA that has the same layout but is defined in plugin.hpp. As some compilers (for instance, Borland C++ 5.5) force 8-byte alignment for structures defined in the Windows API headers, it is not possible to achieve the correct 2-byte alignment for the entire PluginPanelItem structure when the WIN32_FIND_DATA structure is used.

Если определено макро _FAR_USE_WIN32_FIND_DATA, структура PluginPanelItem будет иметь вид, совместимый с FAR 1.65 и FAR 1.70. Однако корректная компиляция плагинов в Borland C++ 5.5 будет невозможна без модификации системных заголовков Windows API.

Если не определено макро _FAR_USE_WIN32_FIND_DATA, будет использоваться структура FAR_FIND_DATA. При этом код будет корректно работать со всеми компиляторами, но некоторые плагины, рассчитанные на то, что используется WIN32_FIND_DATA, не будут компилироваться.

Внимание!

  • В FAR 1.70 release по умолчанию используется вариант, совместимый со старыми плагинами (_FAR_USE_WIN32_FIND_DATA определено). В версии FAR 1.71 умолчанием станет вариант со структурой FAR_FIND_DATA.

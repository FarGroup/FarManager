"Extract files" (Shift-F2)

index

When the user presses Shift-F2 on a file (or on a group of selected files) then FAR carries out the following actions for each of the selected files:

Regular panel

  • opens the file, tries to read a maximum of 0x20000 bytes of data, closes the file;
  • goes though the list of plugins, calling in turn the OpenFilePlugin function;
  • calls the GetFindData function for the plugin that was first to return a value different from INVALID_HANDLE_VALUE and -2 from the OpenFilePlugin function (for the -2 value the search for the right plugin is stopped);
  • calls the GetFiles (for the first file to process OpMode=OPM_TOPLEVEL, for all the remaining files the OPM_SILENT flag is added), in this time, MultiArc for example shows the "Extract" dialog;
  • after a successful call to the GetFiles function FAR clears the selection, calls the FreeFindData function so that the plugin can free the memory allocated by the GetFindData function;
  • closes the plugin by calling the ClosePlugin function.

Plugin panel

if the plugin filled the HostFile field of the OpenPluginInfo structure in a call to the GetOpenPluginInfo function then the actions taken are the same as if the user pressed F5.

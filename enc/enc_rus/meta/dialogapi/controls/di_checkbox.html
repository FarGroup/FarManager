DI_CHECKBOX

главная | Dialog API | Элементы диалога
Элемент диалога DI_CHECKBOX описывает контрольный переключатель (Check Box). также он называется кнопкой с независимой фиксацией (переключатель вкл./выкл., который может переключаться пользователем). Переключатель "включен", если у элемента поле Selected отлично от нуля. 
struct FarDialogItem
{
  int Type          = DI_CHECKBOX
  int X1            = X
  int Y1            = Y
  int X2            = 0 (в Dialog API 1.0 не используется)
  int Y2            = 0 (в Dialog API 1.0 не используется)
  int Focus         = Focus
  int Selected      = Selected
  DWORD Flags       = Flags
  int DefaultButton = DefaultButton
  char Data[512]    = надпись переключателя
};

Внимание!

  • Приведено схематичное изображение структуры; полное описание структуры см. здесь.

Focus

Определяет имеет ли элемент клавиатурный фокус ввода.

Selected

Определяет состояние элемента - отмечен или нет.

Флаги

Для элемента DI_CHECKBOX имеют значения следующие флаги:
ФлагОписание
DIF_CENTERGROUP центрирование идущих подряд переключателей с одинаковой вертикальной позицией (параметр X игнорируется)
DIF_SETCOLOR определить цвет элемента (младший байт параметра Flags должен содержать атрибут текста кнопки)
DIF_DISABLE Элемент является недоступным.
DIF_NOFOCUS Управляющий элемент не получает клавиатурный фокус ввода.
DIF_3STATE Контрольный переключатель будет иметь 3 состояния: "выключен", "включен", "неопределено".
DIF_SHOWAMPERSAND показывать в заголовке рамки амперсанд не используя его для определения горячей клавиши

События

СобытиеОписание
DN_DRAWDLGITEM начало отрисовки текстовой строки
DN_CTLCOLORDLGITEM запрос цветовых атрибутов для отрисовки; Param2: 
LoWord LoByte - текст кнопки
                (COL_WARNDIALOGBUTTON или
                 COL_DIALOGTEXT)
LoWord HiByte - выделенный текст (highlighting text)
                (COL_WARNDIALOGHIGHLIGHTTEXT или
                 COL_DIALOGHIGHLIGHTTEXT)
HiWord LoByte - 0
HiWord HiByte - 0
Если используется заказной атрибут (флаг DIF_SETCOLOR), то текст кнопки (LoWord LoByte) будет тем, что указали.
DN_HOTKEY горячая клавиша (Alt-Символ)
DN_BTNCLICK изменение состояния переключателя
DN_KEY нажали клавишу.
DN_MOUSECLICK "кликнули" клавишей мыши
DN_KILLFOCUS элемент потерял фокус (при условии, что не используется флаг DIF_NOFOCUS).
DN_GOTFOCUS элемент получил фокус (при условии, что не используется флаг DIF_NOFOCUS).

Замечания

Хотя координаты X2 и Y2 в Dialog API 1.0 не используется, но для нормальной работы в будущих версиях FAR желательно корректно заполнить значения этих полей.

DI_VTEXT

главная | Dialog API | Элементы диалога
Элемент диалога DI_VTEXT описывает статическую вертикальную текстовую строку. 
struct FarDialogItem
{
  int Type          = DI_VTEXT
  int X1            = X
  int Y1            = Y1
  int X2            = 0 (в Dialog API 1.0 не используется)
  int Y2            = Y2
  int Focus         = 0
  int Selected      = 0
  DWORD Flags       = Flags
  int DefaultButton = 0
  char Data[512]    = выводимый текст
};

Внимание!

  • Приведено схематичное изображение структуры; полное описание структуры см. здесь.

Флаги

Для элемента DI_VTEXT имеют значения следующие флаги:
ФлагОписание
DIF_SETCOLOR определить цвет элемента (младший байт параметра Flags должен содержать атрибут выводимого текста)
DIF_BOXCOLOR текст будет иметь цвет рамок (COL_DIALOGBOX или COL_WARNDIALOGBOX)
DIF_CENTERTEXT выравнивать текст по центру (учитываются координаты Y1 и Y2).

События

СобытиеОписание
DN_DRAWDLGITEM начало отрисовки текстовой строки
DN_CTLCOLORDLGITEM запрос цветовых атрибутов для отрисовки; Param2: 
LoWord LoByte - текст
                (COL_WARNDIALOGTEXT или
                 COL_DIALOGTEXT
                при выставленном флаге DIF_BOXCOLOR:
                 COL_WARNDIALOGBOX или
                 COL_DIALOGBOX)
LoWord HiByte - 0 (не используется)
HiWord LoByte - 0 (не используется)
HiWord HiByte - 0 (не используется)
Если используется заказной атрибут (флаг DIF_SETCOLOR), то текст (LoWord LoByte) будет тем, что указали.

Замечания

  • Хотя координата X2 в Dialog API 1.0 не используется, но для нормальной работы в будущих версиях FAR желательно корректно заполнить значение этого поля. Укажите то же значение координаты, что и X1.
  • Если указан флаг DIF_CENTERTEXT, то координата Y2 должна быть обязательно указаны, в противном случае действие флага не будет иметь действия.
  • Если не указан флаг DIF_CENTERTEXT, то FAR самостоятельно перерасчитает координаты X2 и Y2, исходя из размера строки и размера диалога.

ProcessName

главная | FarStandardFunctions | CmpName
Функция FSF.ProcessName позволяет обработать некое имя файла: сравнить с маской, с масками или сгенерировать по маске новое имя. 
int WINAPI ProcessName(
  const char *Param1,
  char *Param2,
  DWORD Flags
);

Параметры

Param1
зависит от Flags
Param2
зависит от Flags
Flags
указывает на необходимое действие. Может быть следующим (перечисление PROCESSNAME_FLAGS):
ДействиеОписание
PN_CMPNAME сравнить имя файла с маской. Аналог CmpName: Param1 соответствует Pattern, Param2 соответствует String. В случае успеха возвращается TRUE.
PN_CMPNAMELIST Сравнить имя файла со списком масок, перечисленных через запятую. Аналог предыдущего флага, только Param1 соответствует списку масок. Маски исключения не поддерживаются.
PN_SKIPPATH Может сочетаться с PN_CMPNAME или PN_CMPNAMELIST - указывает на то, что путь к имени файла при сравнении должен игнорироваться.
PN_GENERATENAME генерировать имя файла на основе существующего (Param1) и маски (Param2). Результат записывается в Param2. В случае успеха возвращается TRUE, иначе - FALSE. Если нужно преобразовать только часть Param1, то можно указать размер этой части (может быть до 255 байт) в Flags (добавить по принципу OR), например: Param1="dir1\\file1", а нам нужно изменить только "dir1", тогда параметр Flags должен быть равен PN_GENERATENAME|4

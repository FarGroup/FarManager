29 марта 2006 г.

  • Вернулись обратно к варианту без .CHI-индекса. Если Энциклопедия включена в коллекцию, то не забудьте заменить pluginsr.chi на pluginsr.chm.
  • Изменен дизайн статей Энциклопедии в стиле MSDN (не точная копия, но похоже :-)
  • Bug: Для несколько идущих подряд вызовов ACTL_POSTKEYSEQUENCE срабатывал только первый.
  • Уточнение в InputBox для параметра HelpTopic.
  • MultiArc API: добавлено поле ArcInfo.Chapters - количество глав (chapters) в архиве.
  • MultiArc API: добавлено поле ArcItemInfo.Chapter - глава (chapter) в архиве, которой принадлежит файл.
  • MultiArc API: добавлено описание формата файла custom.ini.
  • Документированы макросы Dlg*.
  • Во время исполнения макросов в ProcessEditorInput передается событие типа FARMACRO_KEY_EVENT.
  • Сообщение DM_LISTSETTITLES не работает, если в листе ни одного элемента.
  • Дублирование курсоров после удаления пунктов списка DI_LISTBOX в случае, если курсор стоит не на удаляемом элементе.
  • Пара новых сообщений DM_GETSELECTION и DM_SETSELECTION, позволяющих управлять выделением блоков в строках редактирования диалога.
  • Новые команды функции AdvControl: ACTL_GETPLUGINMAXREADDATA и ACTL_GETWCHARMODE.
  • Флаг FRS_SCANSYMLINK позволяет функции FSF.FarRecursiveSearch "заходить" в символические связи во время обхода дерева каталогов.
  • Некорректное центрирование элементов DI_RADIOBUTTON и DI_CHECKBOX с выставленным флагом DIF_CENTERGROUP.
  • Теперь FSF.GetReparsePointInfo для сетевых каталогов возвращает ошибку, т.к. корректную информацию о содержимом символической связи в этом случае получить невозможно.
  • В связи с выносом диалоговых настроек отдельным пунктом добавлена команда ACTL_GETDIALOGSETTINGS. Соответственно изменены значения и названия следующих констант:
    БылоСтало
    FIS_PERSISTENTBLOCKSINEDITCONTROLS FDIS_PERSISTENTBLOCKSINEDITCONTROLS
    FIS_HISTORYINDIALOGEDITCONTROLS FDIS_HISTORYINDIALOGEDITCONTROLS
    FIS_AUTOCOMPLETEININPUTLINES FDIS_AUTOCOMPLETEININPUTLINES
  • FAR возвращал мусор в PanelInfo.SelectedItems[0] для случая, когда на панели нет выделенных файлов и курсор стоит на элементе ".."
  • Все перечисления в plugin.hpp сделаны именованными
  • Новая команда MCMD_POSTMACROSTRING - передать в FAR макрос в текстовом виде (в том виде, как их хранит FAR Manager в реестре)
  • Уточнение структуры ActlKeyMacro: добавлен член Param.PlainText - указатель на строку, содержащую макропоследовательность.
  • Новый флаг KSFLAGS_NOSENDKEYSTOPLUGINS - не передавать клавиши редакторным плагинам (плагинам, экспортирующим функцию ProcessEditorInput)
  • Новая команда ESPT_SETWORDDIV - позволяет изменить строку символов-разделителей слов (параметр cParam) для текущей копии редактора.
  • Новая команда ESPT_GETWORDDIV - позволяет получить строку символов-разделителей слов (параметр cParam) для текущей копии редактора.
  • Bug: Если заголовок диалога начинается с числа, то это число отрезается в заголовке окна Фара.
  • Bug: Смена фокуса (нажатие клавиши Tab или Shift-Tab) в диалоге с единственным элементом приводила к зависанию FAR.
  • Поле FarDialogItem.Data для DI_LISTBOX используется как верхний заголовок списка (если не выставлен флаг DIF_LISTNOBOX).
  • В качестве заголовка консоли принимается заголовок списка (DI_LISTBOX), если в диалоге нет рамок и список (DI_LISTBOX) первый элемент.
  • Bug: Преобразование 'Key = FarNameToKey ("AltCtrlБуква") -> FarKeyToName (Key)' работало некорректно (возвращается Alt00000, вместо AltCtrlБуква)
  • Bug: В .lng файле нельзя было ставить пробелы после идентификатора ".Language", например ".Language = foo".
  • Bug: FSF.FarRecursiveSearch, не может найти файл если в качестве маски задано его короткое имя.
  • Новый флаг FMENU_CHANGECONSOLETITLE функции Menu. Если указан, то FAR изменит заголовок консоли в значение, указанное в параметре Title (если Title не пуст).
  • Новый флаг DIF_LISTNOCLOSE - при выборе элемента из списка не закрывать диалог.
  • Сообщение DM_LISTSETMOUSEREACTION теперь позволяет более гибко управлять слежением списка за передвижением курсора мыши (перемещение курсора в списке). С помощью этого сообщения можно задавать одно из трех состояний:
    1. список будет реагировать на движение мыши только если элемент в фокусе,
    2. список будет всегда реагировать на движение мыши
    3. список не будет реагировать на движение мыши
  • При вызове команды ECTL_SAVEFILE FAR не запоминал новое имя файла.
  • Пара новых статей от Eugene Yavorsky: Плагин для разных версий FAR Manager и Как уменьшить размер плагина: Microsoft VC++ 4.2 (замечания к статье Олега Данилова)
  • Новое сообщение DN_LISTHOTKEY. Сообщает обработчику диалога, что пользователь для смены пункта в списке (DI_LISTBOX) воспользовался горячими клавишами.
  • Bug: При движении мышью по листу сообщение DN_LISTCHANGE посылалось даже, если элемент списка не менялся. т.е. если двигать мышью в пределах одного элемента, будут посылаться много DN_LISTCHANGE с номером этого элемента.
  • Логическое дополнение к числовой сортировке: команды FCTL_SETNUMERICSORT и FCTL_SETANOTHERNUMERICSORT флаг PFLAGS_NUMERICSORT (включен NumericSort на панели) к PanelInfo.Flags
  • Bug: Падение FAR, если в функции FSF.PointToName() параметр Path == NULL.
  • Bug: При сохранении файла в редакторе, вызванного из плагина портился пользовательский заголовок, переданный в параметре Title функции Editor.
  • Опубликован Viewer API.
  • Добавлен раздел Panel API.
  • Флаг EOPT_EXPANDTABS переименован в EOPT_EXPANDALLTABS.
  • Добавлен раздел, описывающий макросы.
  • Статья Алексея Ярославского: Как компилировать с помощью Borland C++ 5.5 без startup code
  • Новый флаг ECF_TAB1. Начиная с build 1957 FAR раскрашивает символ табуляции на всю длину, на которую расширяется табуляция. Если необходимо, чтобы символ табуляции для цветовой области занимал 1 знакоместо, то для элемента EditorColor.Color, в дополнении к цвету, необходимо выставить флаг ECF_TAB1.
  • ECTL_GETBOOKMARKS и ECTL_GETINFO можно вызывать в EE_CLOSE.
  • В API поддержки архивов добавился флаг AF_HDRENCRYPTED, показывающий, что у архива шифрованный каталог.
  • Замечания к статье "Плагин для разных версий FAR Manager", по поводу сравнения версий.
  • Начиная с 1.70 build 2052 для панельных плагинов, использующих функцию "ProcessKey", снято ограничение на приходящие в функцию клавиши.
  • Добавлена макрообласть "UserMenu"
  • Новый флаг PKF_PREPROCESS.

09 апреля 2003 г.

  • Теперь Энциклопедия распространяется в виде двух файлов pluginsr.chi (индекс) и pluginsr.chm ("данные")
  • Имена таблиц, возвращаемые плагинам через функцию CharTable, не соответствовали именам, которые показывались в списке по Shift-F8.
  • Bug: FAR изменял переданные плагином данные для панели "Информация" (Ctrl-L).
  • Второй параметр у экспортируемой фунцкии SetFindList является const.
  • Во время вызова ExitFAR плагину недоступны следующие сервисные функции: InputBox, Menu, Dialog, DialogEx GetMsg, Message, Control, SaveScreen, RestoreScreen, GetDirList, Viewer, Editor, CharTable, Text, EditorControl, GetPluginDirList, ShowHelp, AdvControl (частично).
    В AdvControl доступны только следующие команды: ACTL_GETFARVERSION, ACTL_GETSYSWORDDIV, ACTL_GETCOLOR, ACTL_GETARRAYCOLOR, ACTL_GETFARHWND, ACTL_GETSYSTEMSETTINGS, ACTL_GETPANELSETTINGS, ACTL_GETINTERFACESETTINGS, ACTL_GETCONFIRMATIONS, ACTL_GETDESCSETTINGS.
  • Запрещено создание немодального редактора или просмотрщика из модального.
  • Запрещено переключение из модального в другое окно.
  • ESPT_LOCKMODE - плагины могут запретить/отменить запрет изменения файла в редакторе.
  • ECTL_TURNOFFMARKINGBLOCK - сбросить флаг редактора, который устанавливается всегда, когда пользователь осуществляет выделение блока.
  • Некорректное отображение списка DI_LISTBOX с флагом DIF_LISTNOBOX
  • В списке DI_LISTBOX не обрабатывались горячие клавиши.
  • В диалоге при активном фокусном списке DI_LISTBOX не обрабатывались горячие клавиши.
  • При получении фокуса ввода неверно отрисовывался элемент DI_EDIT с выставленным флагом DIF_SELECTONENTRY.
  • Некорректно формировался длинный заголовок диалога.
  • Некорректно обрабатывалось программное переключение (DM_SETCHECK) для группы зависимых кнопок (DI_RADIOBUTTON).
  • Цвет разделительной линии в диалогах не соответствовал цвету рамки.
  • Новый флаг LIF_DELETEUSERDATA, который заставляет FAR удалить ассоциированные данные при обновлении пункта списка командой DM_LISTUPDATE. По умолчанию данные при обновлении не удаляются.
  • DM_LISTADDSTR возвращал некорректное значение позиции добавляемого пункта в списке.
  • При обновлении списка (DM_LISTUPDATE) прикрепленные пользовательские данные (DM_LISTSETDATA) пункта списка удалялись.
  • DM_SETTEXT (DM_SETTEXTPTR) теперь работают с DI_LISTBOX. Происходит обновление текущего элемента списка. Прикрепленные пользовательские данные (DM_LISTSETDATA) пункта списка не изменяются.
  • Новый флаг FIB_NOAMPERSAND - Не показывать в приглашении ввода амперсанд, использовать его для определения горячей клавиши.
  • Константы CONSOLE_* переименованы в FAR_CONSOLE_* (конфликт с константами из WinCon.h)
  • Некорректная отрисовка текущего указателя в меню для случая, когда в переданном массиве пунктов меню первым элементом стоял разделитель и не был задан текущий элемент (или в качестве текущего элемента был сам разделитель).
  • Экспортируемая функция PutFiles может возвращать число 2 (успех, но позиционировать курсор на файл не нужно).
  • Изменилось описание структуры OpenPluginInfo - некоторые члены стали const
  • Конструкция Message(FMSG_ALLINONE,"\nFoobar") приводила к падению FAR (первым символом '\n' пытаемся вывести диалог сообщения без заголовка)
  • FCTL_GETPANELSHORTINFO
    Аналогична FCTL_GETPANELINFO, но поля PanelItems и SelectedItems структуры PanelInfo не заполнены и равны NULL. Эта команда предназначена для получения общей информации о панели без конкретного знания файлового контента.
  • FCTL_GETANOTHERPANELSHORTINFO
    Аналогична FCTL_GETPANELSHORTINFO, но получает информацию о пассивной панели.
  • Новая команда FCTL_CHECKPANELSEXIST. Позволяет проверить доступность панелей. В режиме, когда FAR запущен с ключами /e или /v (в качестве внешнего редактора или программы просмотра), панели не создаются.
  • Некорректное отсечение длинного заголовка (параметр Title) в функции Message.
  • Два новых флага VF_IMMEDIATERETURN и EF_IMMEDIATERETURN, позволяющих плагину получать управление сразу же после открытия редактора/программы просмотра. Реально редактор/программа просмотра начнёт работать после возвращения плагином управления ФАРу. Может быть использовано для открытия нескольких редакторов/программ просмотра мгновенно.
  • Функция FSF.MkLink некорректно работала при создании симолической связи, если в качестве источника был полный путь к каталогу.
  • Функция FSF.MkLink самостоятельно обновляет панели, если не указан флаг FLINK_DONOTUPDATEPANEL.
  • Новый флаг DIF_CENTERTEXT для статических элементов DI_TEXT и DI_VTEXT.
  • VF_DELETEONLYFILEONCLOSE и EF_DELETEONLYFILEONCLOSE - аналоги VF_DELETEONCLOSE и EF_DELETEONCLOSE, но удаляется только файл, а папка, содержащая его, не удаляется, даже если она пуста.
  • Ctrl-Ins в строках ввода сбрасывал состояние UnChanged.
  • Падение в обработчике ECTL_SETSTRING, если поле EditorSetString.StringLength содержало отрицательное значение.
  • Новая команда ECTL_DELETEBLOCK - удалить блок в редакторе.
  • Если у DI_SINGLEBOX (DI_DOUBLEBOX) координаты X1==X2 или Y1==Y2, то рисуется соответствующая линия (вертикальная или горизонтальная)
  • Если у DI_SINGLEBOX (DI_DOUBLEBOX) указан флаг DIF_SETCOLOR, то младший байт позволяет задавать цвет рамки.
  • Два новых флага для DialogEx: FDLG_NODRAWSHADOW - не рисовать тень диалога, FDLG_NODRAWPANEL - не рисовать поле диалога
  • При указании флага FDLG_SMALLDIALOG поле диалога теперь отрисовывается.
  • Новая команда ACTL_SETARRAYCOLOR, позволяющая изменить цветовую схему FAR Manager.
  • Константы COL_DIALOGMENU* заменен на COL_DIALOGLIST*
  • Измененно сообщение DN_CTLCOLORDLGLIST. Теперь Param1 указывает на номер элемента диалога (типа DI_LISTBOX, DI_COMBOBOX или строковый редактор c историей ввода). Param2 - указатель на структуру FarListColors, описывающую цветовую схему списка.
  • Теперь заголовок элемента DI_LISTBOX описывается цветом COL_DIALOGLISTTITLE
  • По умолчанию цвет для индекса COL_DIALOGLISTDISABLED - темносерый на светлосером.
  • Немного добавок в разделе Frequently Asked Questions.
  • Не прорисовывался элемент DI_LISTBOX с флагом DIF_LISTNOBOX высотой в 1 строку..

20 марта 2002 г.

  • Вместо FARMANAGERVERSION в функции GetMinFarVersion() можно воспользоваться макросом MAKEFARVERSION
  • Изменилось определение структуры FarDialogItem для того, чтобы plugin.hpp можно было использовать с pure C компиляторами.
  • Изменена логика возвращаемого значения на сообщение DN_BTNCLICK для элементов DI_BUTTON. Теперь это:
    Для DI_BUTTON:
    1. TRUE - сообщение обработано, дальше не продолжать обработку этого сообщения ядром.
    2. FALSE - продолжить обработку ядром.
    Для остальных элементов:
    1. TRUE - разрешить изменение (и соответственно отрисовку того, что пользователь ввёл)
    2. FALSE - запретить изменение.
  • Новый флаг EJECT_LOAD_MEDIA для ACTL_EJECTMEDIA. Работает только в NT/2000/XP
  • DIF_SHOWAMPERSAND также обрабатывается для DI_BUTTON, DI_CHECKBOX, DI_RADIOBUTTON
  • После выбора строки из истории в функцию обработки диалога не посылалось сообщение DN_EDITCHANGE.
  • Изменена реакция стандартного обработчика диалога на событие DN_MOUSECLICK за пределами диалога. Теперь, если функция диалога не обрабатывает событие DN_MOUSECLICK с Param1=-1, то диалог будет закрыт.
  • Уточнения в структуре PanelInfo - поле Flags. Также добавлены флаги PANELINFOFLAGS.
  • В ProcessHostFile не передавался флаг OPM_TOPLEVEL в том случае, когда это требовалось (нажатие Shift-F3 на архиве в файловой панели).
  • Некорректно работало сообщение DM_SETMAXTEXTLENGTH.
  • Две новых "клавиши" для колеса мыши - KEY_MSWHEEL_UP/KEY_MSWHEEL_DOWN.
  • Сообщения диалога для работы со списками (DI_LISTBOX и DI_COMBOBOX) - DM_LISTSORT, DM_LISTGETITEM, DM_LISTGETCURPOS, DM_LISTSETCURPOS, DM_LISTDELETE, DM_LISTADD, DM_LISTUPDATE, DM_LISTINFO, DM_LISTFINDSTRING, DM_LISTINSERT, DM_LISTSETTITLE, DM_LISTGETTITLE... и соответствующие им структуры.
  • Сообщения DM_GETDROPDOWNOPENED DM_SETDROPDOWNOPENED, DM_SETHISTORY, DM_SETITEMPOSITION.
  • Новые флаги VF_ENABLE_F6 (в функции Viewer) и EF_ENABLE_F6 (в функции Editor), разрешающие переключение по F6 из вьюера и редактора, открытых при помощи соответствующих функций.
  • Флаг DIF_CENTERGROUP имеет смысл только для следующих элементов: DI_BUTTON, DI_CHECKBOX, DI_RADIOBUTTON и DI_TEXT.
  • Новый цвет COL_MENUDISABLEDTEXT - недоступный пункт меню.
  • DM_RESIZEDIALOG - изменить размеры диалога.
  • DM_GETCHECK/DM_SETCHECK/DM_SET3STATE/BSTATE_* - работа с состоянием элементов DI_RADIOBUTTON/DI_CHECKBOX
  • Функция GetMinFarVersion не вызывалась для плагинов с флагом PF_PRELOAD, а также для плагинов, имеющихся в кэше.
  • Теперь функция Viewer возвращает FALSE при ошибке открытия и в том случае, когда вьюер открывается с флагом VF_NONMODAL.
  • Новая FSF-функция MkLink. Умеет создавать хардлинки, символические связи и монтировать диски на файловую систему.
  • Во время исполнения макроса в меню игнорировались значения массива BreakKeys, переданных в функцию Menu.
  • Функция Unquote убирает все двойные кавычки из строки.
  • Плагину в функцию GetFiles в качестве параметра DestPath передавалось имя каталога, усеченного до первой точки. Это по поводу: "Надоело распаковывать some.foo.rar в каталог some\"
  • Функция FSF.ExpandEnvironmentStr манипулирует строками в OEM-кодировке.
  • DN_DRAGGED - информация о процессе перемещения диалога.
  • ACTL_GETWINDOWCOUNT, ACTL_GETWINDOWINFO, ACTL_SETCURRENTWINDOW и ACTL_COMMIT, struct WindowInfo - манипулирование "окнами" FAR Manager
  • ECTL_GETBOOKMARKS, struct EditorBookMarks - информация о закладках в редакторе (EditorInfo.BookMarkCount - количество закладок, поддерживаемых редактором).
  • Удален флаг LIF_PTRDATA - изжил себя как класс (соответственно изменилась структура FarListItem)
  • ACTL_POSTKEYSEQUENCE - поместить последовательность клавиш для последующего выполнения (когда FAR Manager получит управление).
  • ACTL_GETFARHWND - возвращает HWND текущего окна FAR.
  • ACTL_GET*SETTINGS - команды, позволяющие плагину узнать некоторые настройки FAR Manager.
  • DM_EDITUNCHANGEDFLAG - позволяет управлять состоянием "неизменный текст" для полей ввода.
  • ECTL_READINPUT: во время исполнения макроса плагину возвращается корректно заполненная структура INPUT_RECORD.
  • Функция FSF.ConvertNameToReal преобразует относительное имя файлового объекта в полное с учетом символических связей (т.н. reparse point в Windows 2000) и текущего каталога.
  • Все поля структуры CharTableSet кроме TableName имеют тип unsigned char.
  • Дополнительное поле в структуре PluginPanelItem - CRC32.
  • Опциональная посылка DN_MOUSEEVENT в обработчик диалога. По умолчанию отключена (DN_MOUSEEVENT/DM_SETMOUSEEVENTNOTIFY)
  • FCTL_GETCMDLINESELECTION, FCTL_SETCMDLINESELECTION, FCTL_GETCMDLINESELECTEDTEXT.
  • GetPluginDirList: если параметр hPlugin равен INVALID_HANDLE_VALUE, то значение параметра PluginNumber игнорируется (используется номер плагина, которому принадлежит активная панель).
  • DM_GETITEMPOSITION - получить позицию и размеры управляющего элемента.
  • Плагины могут при помощи ECTL_SETPARAM менять текущую кодировку в редакторе.
  • DN_RESIZECONSOLE - уведомляющее сообщение "Изменились размеры консоли".
  • DM_GETTEXTLENGTH возвращает размер строки без учета '\0'
  • Формат HelpTopic для меню, диалогов и сообщений может быть:
    1. ":Topic"
      вызов главной темы (из основного файла помощи FAR)
    2. "Topic"
      вызов локальной темы
    3. "<FullModuleName>Topic"
      вызов темы помощи из каталога, где находится модуль "FullModuleName"
    4. "<FullPath>Topic"
      вызов темы помощи из каталога "FullPath" Здесь "FullPath" путь ДОЛЖЕН заканчиваться символом '\'
    Если для случая 3 и 4 параметр Topic не указан, то будет подставлен "Contents"
  • Усовершенствование FarRecursiveSearch:
    1. Проверяется правильность параметров.
    2. Теперь обработка каталогов не зависит от маски файлов.
    3. Маска может быть стандартного ФАРовского вида (со скобками, перечислением и пр.). Может быть несколько масок файлов, разделенных запятыми или точкой с запятой, можно указывать маски исключения, можно заключать маски в кавычки.
  • Теперь DI_LISTBOX с выставленным флагом DIF_LISTNOBOX и меню с флагом LINFO_SHOWNOBOX рисуются без лишних окантовывающих пробелов вокруг списка, что автоматически позволяет использовать DI_LISTBOX в диалогах, не опасаясь затирания рамки самого диалога пустым местом от списка.
  • FMENU_USEEXT, MIF_*, FarMenuItemEx - "расширение" меню.
  • При создании нового файла также посылается событие EE_READ плагинам.
  • Новая функция FSF.GetReparsePointInfo - позволяет для указанной символической связи получить информацию об источнике - путь к исходному каталогу/диску. Функция работает только в Windows 2000 и выше. В отличие от ConvertNameToReal, применима только для симлинков.
  • FSF.FarInputRecordToKey: Параметр имеет модификатор "const".
  • Элемент диалога DI_USERCONTROL некорректно работал с курсором.
  • Устранено падение FAR Manager, происходившее в случае посылки сообщения DM_SETFOCUS в ответ на событие DN_GOTFOCUS.
  • В HLF-файлах можно указывать размер табуляции, вставив в начало файла помощи конструкцию ".Options TabSize=Размер". "Размер" может быть от 1 до 16, по умолчанию - 8.
  • Неверно рассчитывались координаты элементов DI_TEXT и DI_VTEXT.
  • EditorGetString.StringText и EditorGetString.StringEOL объявлены как const.
  • EditorInfo.CurState - состояние редактора (дополнительные флаги-ECSTATE_*)
  • Флаг EF_DELETEONCLOSE. Аналогичен VF_DELETEONCLOSE.
  • FarListInfo и флаги LINFO_*
  • DIF_SEPARATOR2 - двойная разделительная линия.
  • Неверно обрабатывалась команда ECTL_QUIT.
  • Если указан флаг DIF_LISTWRAPMODE, то попытка перемещения курсора в списке выше первого пункта или ниже последнего будет приводить к перемещению соответственно к последнему или к первому пункту.
  • Флаг EF_DISABLEHISTORY для функции Editor - запретить добавление имени файла в историю редактирования (Alt-F11). По умолчанию имя в историю добавляется.
  • FarListItem: размер текста 128 + DWORD Reserved
  • Флаги диалога DialogEx:
    FDLG_WARNING - задает для диалога цветовую схему "Предупреждение". FDLG_SMALLDIALOG - позволяет создавать диалоги с уменьшенным размером рамки. Для таких диалогов не рисуется поле диалога, тень, а при отрисовке разделительных линий (DIF_SEPARATOR) предполагается нулевой отступ между границей диалога и окружающей его двойной рамкой.

ProcessName

главная | FarStandardFunctions | CmpName
Функция FSF.ProcessName позволяет обработать некое имя файла: сравнить с маской, с масками или сгенерировать по маске новое имя. 
int WINAPI ProcessName(
  const wchar_t *Param1,
  wchar_t *Param2,
  DWORD Size,  
  DWORD Flags
);

Параметры

Param1
зависит от Flags
Param2
зависит от Flags
Size
зависит от Flags
Flags
указывает на необходимое действие. Может быть следующим (перечисление PROCESSNAME_FLAGS):
ДействиеОписание
PN_CMPNAME Сравнить имя файла с маской. Аналог CmpName: Param1 соответствует Pattern, Param2 соответствует String, Size игнорируется. В случае успеха возвращается TRUE.
PN_CMPNAMELIST Сравнить имя файла со списком масок, перечисленных через запятую. Аналог предыдущего флага, только Param1 соответствует списку масок. Маски исключения не поддерживаются. Size игнорируется.
PN_SKIPPATH Может сочетаться с PN_CMPNAME или PN_CMPNAMELIST - указывает на то, что путь к имени файла при сравнении должен игнорироваться.
PN_GENERATENAME Генерировать имя файла на основе существующего (Param1) и маски (Param2). Результат, длиной Size, записывается в Param2 (учитывайте, что размер буфера Param2 должен быть не менее Size). Возвращается функцией требуемый размер буфера Param2.
Если нужно преобразовать только часть Param1, то можно указать размер этой части (может быть до 255 байт) в Flags (добавить по принципу OR), например: Param1="dir1\\file1", а нам нужно изменить только "dir1", тогда параметр Flags должен быть равен PN_GENERATENAME|4

ConvertPath

главная | FarStandardFunctions
Функция FSF.ConvertPath преобразует относительное имя файлового объекта в полное.

Внимание!

  • Не используйте для этих целей функцию GetFullPathName из Windows API.

    Не передавайте относительные имена файлов в любые функции Windows API, все относительные пути должны быть преобразованы в полные. При передаче имён в функции Windows API используйте префикс "\\?\", чтобы избежать проблем с длинными и нестандартными именами файлов. 
int WINAPI ConvertPath(
  CONVERTPATHMODES Mode,
  const wchar_t *Src,  
  wchar_t *Dest,
  int DestSize
);

Параметры

Mode
Указывает на необходимое действие. Может быть следующим (перечисление CONVERTPATHMODES):
ДействиеОписание
CPM_FULL преобразует относительное имя файлового объекта в полное.
CPM_REAL преобразует относительное имя файлового объекта в полное, с учетом символических связей (т.н. reparse point в Windows 2000).
Src
Источник данных - полное или относительное имя файла/каталога.
Dest
Строка назначения, сюда будет помещен полный путь. Может быть равна NULL.
DestSize
Размер принимаемых данных. Если Dest=NULL, то параметр DestSize игнорируется.

Возвращаемое значение

Реальный размер полного пути, размещенного в Dest или требуемый размер строки для случая, когда Dest=NULL.
  • Для монтированных дисков, которые не имеют буквы диска, функция поместит в Dest строку, подобную этой: "\\?\Volume{273872e0-5e49-11d5-b614-0080ad70bb9b}\Foo.bar"
  • Если, например, каталог "D:\Foo\Bar" является симлинком для реально существующего "C:\work\Doc", то функция для файла "D:\Foo\Bar\1092\readme.txt" вернет "C:\work\Doc\1092\readme.txt"

Примечание

  1. Параметры Src и Dest могут ссылаться на одну и ту же строку.

Пример

Неправильно:

GetFileAttributes(L"..\some_file");

Правильно:

   size_t Size=FSF.ConvertPath(CPM_FULL,L"..\some_file",NULL,0);
   WCHAR FullName=new WCHAR[Size];
   FSF.ConvertPath(CPM_FULL,L"..\some_file",FullName,Size);
   GetFileAttributes(FullName);
   delete[] FullName;

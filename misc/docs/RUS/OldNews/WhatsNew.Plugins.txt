FAR 1.70 (build 2087) (29.03.2006)
---------------------------------------------------------------------------
   Все поставляемые вместе с FAR Manager плагины предназначены только для
той версии FAR,  в дистрибутив которой они входят. Корректная работа этих
плагинов с другими версиями Фара не гарантируется.

MultiArc
~~~~~~~~
[+] Добавлена возможность управлять приоритетом процесса архивирования.
[*] При просмотре архивов ARJ показывается информация о времени создания
    файла и времени последнего доступа к нему, если архив создан версией
    ARJ, которая поддерживает сохранение этой информации.
[*] В панели информации показывается количество глав (Chapters) в архиве.
[+] В синтаксис custom.ini добавлен метасимвол 'C'. При анализе листинга
    архиватора будет заменен на количество Chapter`ов.
[!] CAB.FMT теперь имеет настройки по умолчанию, рассчитанные на работу с
    MsCab.exe версии 0.63 и выше. Последние версии архиватора можно взять
    здесь http://plugring.farmanager.com/cgi-bin/downld.cgi?Draw=List&
    Select=PlugIn&SelectPlugIn=310
[-] Некорректно инициализировались модули второго уровня перед добавлением
    файлов в архив с включенной опцией "Читать архивные описания".
[-] В процессе анализа файловой структуры архива некорректно выставлялись
    атрибуты для каталогов (hidden, readonly).
[-] Битые архивы, не содержащие файлов, открывались с повторным сообщением
    об ошибке.
[*] Во все FMT-модули добавлены ресурсы версий.
[+] Добавлен модуль ARC.FMT
[!] Ужесточен механизм распознавания lha, rar и zip архивов. Некоторые
    файлы (например, модули KasperskyLab - minizip.ppl, rar.ppl, zip.ppl)
    содержат сигнатуры для распознавания архивов. Именно эти сигнатуры
    и были причиной того, что MultiArc считал такие файлы архивами.
[-] Некорректно обрабатывался конец RAR-архива. RAR.FMT считал некоторые
    архивы некорректными и выдавал сообщение "Неожиданный конец архива"
    ("Unexpected end of archive").
[-] Некорректно обрабатывались некоторые поля LHA-архивов (например, поле
    файловых атрибутов).
[-] TARGZ.FMT считал некорректными TAR-архивы, созданные некоторыми версиями
    tar под AIX,
[-] Неправильная интерпретация директории TAR-архивов.
[!] Устранена несовместимость с технологией "Data Execution Prevention",
    которая имеется в некоторых современных процессорах и поддерживается
    начиная с Windows XP SP2 и Windows Server 2003 SP1.
[!] В custom.ini в качестве значений параметров 'FormatX', 'IgnoreStringX',
    'Start', 'End' можно использовать регулярные выражения с синтаксисом
    Perl5. Описание нового синтаксиса можно прочитать в custom.ini.
[+] Префикс 'ma' для открытия архива на прямую в плагине.
[+] Для получения каталога RAR-архива с шифрованным заголовком используется
    UnRAR.dll (если библиотека доступна по путям). Из-за ограничения АПИ
    библиотеки UnRAR.dll поля даты создания и последнего доступа будут пусты.
[+] Добавлен флаг AF_HDRENCRYPTED.
[+] TechInfo#71.

Network
~~~~~~~
[-] Дублирование скрытых ресурсов при переходе на несуществующий путь по
    команде 'CD' в панели плагина.
[-] Некорректная обработка двойных кавычек в префиксе: команда
      net:"server"
    обрабатывалась буквально - производился поиск машины с именем "server"
[-] Неправильное поведение плагина при попытке открыть удаленный ресурс,
    когда используется неверная учетная запись, или, когда пользователь
    прерывает процедуру логина.
[-] Многократное сканирование сети при неудачной попытке логона.
[+] Добавлена возможность открывать удаленные ресурсы с прямым указанием
    данных учетной записи. Смотрите справку.
[-] Неправильное поведение при переходе по дереву на уровень выше из
    ресурса, содержащего в своем имени символы отличные от английских.
[-] При подключении из командной строки к удаленному компьютеру, доступ к
    которому текущему пользователю закрыт, плагин не выдавал никаких
    сообщений и показывал пустую панель. Теперь при подключении к такому
    компьютеру плагин запрашивает имя пользователя и пароль, в случае
    отказа пользователя, переходим в список провайдеров.
[+] Изменен алгоритм подключения к удаленному ресурсу с прямым указанием
    данных учетной записи. Теперь, если в ответ на попытку логина система
    возвращает ошибку "Неверное имя пользователя или пароль", плагин
    попытается привести имя пользователя к виду "Computer\User", где
    Computer - имя удаленного компьютера, а User, соответственно, имя
    пользователя. Затем будет произведена повторная попытка подключения,
    если же и на этот раз произойдет ошибка, плагин уведомит пользователя
    соответствующим сообщением.
[-] При работе в системах Windows 95/98/ME из-за внутренней ошибки системы
    иногда создавались ситуации, когда любая попытка обращения к плагину
    заканчивалась выдачей сообщения 'cannot browse network'.
[-] При попытке доступа к сетевому ресурсу, представляющему собой корень
    пустого диска, выдавалась ошибка "Файл не найден".
[+] В список провайдеров добавлена виртуальная папка "Избранное".
[+] Добавлена опция "Запомнить пароль" в диалоге логина. Если данная опция
    выбрана, в случае успешного подключения к удаленному ресурсу, данные
    учетной записи, использованной для логина, будут сохранены в реестр,
    а ресурс будет автоматически добавлен в папку "Избранное".
[!] Замечание, касающееся безопасности. Данные учетных записей,
    используемых для подключения к ресурсам из папки "Избранное", хранятся
    в реестре в ОТКРЫТОМ виде.
[+] Изменен алгоритм подключения к ресурсам:
    1. Плугин пытается подключиться к ресурсу, не указывая ни имени
    пользователя, ни пароля. Если попытка завершилась неудачей, переходим
    к пункту 2.
    2. Плугин пытается найти в папке "Избранное" указанный ресурс. Если
    ресурс найден, пытается подключиться, используя соответствующие данные из
    реестра. Если же опять не удалось подключиться, переходим к пункту 3.
    3. Пользователь вводит имя и пароль, плугин пытается подключиться к
    ресурсу используя указанные данные.
[+] В настройках плугина добавлена опция "Показывать полные пути к общим
    ресурсам", по-умолчанию включена. Если эта опция включена, плугин
    отображает пути как и раньше (пример: "\\Server", "\\Server\Share"),
    если отключена, то пути показываются в сокращенном виде
    (пример: "Server", "Share").
[+] В диалог настроек добавлена опция "Возврат в Избранное, если ресурс в
    избранном", по-умолчанию включена. Если опция включена, то нажимая на
    ".." попадем в Избранное, если ресурс находится в избранном. Если
    отключена, то проверка, есть ли ресурс в избранном не производится.
[+] В диалог настроек добавлена опция "Проверка доступности ресурсов",
    по-умолчанию включена. Если опция включена, то в "Избранном" будут
    показаны только ресурсы, находящиеся в данный момент в сети.
[+] В диалог настроек добавлена опция "Выводить ".." в списке провайдеров",
    по-умолчанию включена.
[*] При вызове из командной строки плагин преобразовывает имя файла в
    верхний регистр.
[+] Добавлена возможность создания папок в избранном.

[+] ACE.fmt - модуль поддержки архивов ACE для FAR. Из-за некоторой
    несовместимости разных версий архиватора ACE, некоторые архивные
    команды могут работать некорректно. Это происходит из-за новых
    ключей, появившихся в новой версии (на данный момент версия 1.2a).
    Если кто-нибудь будь получит сообщение об ошибке при работе с архивом,
    настройте параметры командных строк в меню: Параметры|Параметры внешних
    модулей|Работа с архивами|Командные строки|ACE и уберите где найдёте
    параметр {-t%%W}.


TmpPanel
~~~~~~~~~
[+] Временная панель может считывать список элементов из файла-
    списка. Подробности смотрите в помощи к плагину.
[+] Префикс - "tmp".


PrintMan
~~~~~~~~
[+] Новый плагин. Умеет печатать из программы просмотра, редактора и
    из панелей. Инсталлировать новые сетевые принтеры...
    Умеет печатать на те принтеры на которые не умеет печатать встроенная
    программа (или подпрограмма? :-) печати и подаёт formfeed после печати.
    Подробности смотрите в помощи к плагину.


FARCmds
~~~~~~~
[+] Новый плагин, предоставляет дополнительные возможности, которые
    можно использовать в панелях, меню пользователя, ассоциаций файлов или
    командной строке, а именно: открытие нужного файла в редакторе/программе
    просмотра FAR'а, перехват stdout'а какой-либо программы с последующим
    его показом в редакторе/программе просмотра, "умный" переход к нужному
    каталогу/файлу, а также установление на пассивной панели такого же
    каталога, что и на активной.
    Подробности смотрите в помощи к плагину.


Brackets
~~~~~~~~
[+] Добавлена обработка угловых скобок '<' и '>'


EditCase
~~~~~~~~
[!] Полностью переписан и умеет обрабатывать текст в следующих режимах:

       * Начинать С Прописных

       * иЗМЕНИТЬ рЕГИСТР

[!] Без выделения преобразует текущее слово.

FAR 1.70 (build 2087) (29.03.2006)
---------------------------------------------------------------------------
Основное
~~~~~~~~
[!] Все сопроводительные русскоязычные текстовые документы перекодированы
    из OEM в ANSI кодировку.
[!] Ключ /aw удален. В Windows NT4 и выше по умолчанию FAR запускается в
    Unicode-режиме консоли. Для работы в ANSI-режиме предусмотрен ключ
    командной строки /8.
[!] Изменения и уточнения в Plugin API подробнее см. в WhatsNew.Api.txt.
[!] Запуск "старых" версий FAR (ниже 1.70 build 1784), с выставленным
    автосохранением (или принудительное сохранение настроек - Shift-F9),
    приведет к потере "новых" макросов (в реестре), т.к. начиная с
    build 1784 изменена логика поведения макродвижка.
     a. при чтении макропоследовательности из реестра кроме компиляции
        так же запоминается исходный текст макроса.
     b. декомпилится только простой макрос (встретив код спец.макроклавиши
        декомпиляция прерывается и возвращается дубликат исходного текста
        макропоследовательности) и только в момент запоминания
        введенной последовательности (когда назначили макроклавишу).
     c. при сохранении макрос не декомпилится, а берется исходный текст
        макроса; декомпилится только простой макрос, т.е. в котором есть
        ТОЛЬКО "чистые" клавиши (без всяких там '$что-то'), такие, какие
        были в 1.65 и ниже.

Система
~~~~~~~
[-] По умолчанию был выбран неверный режим показа имен файлов на панели
    ("Показывать имена файлов строчными буквами" вместо "Показывать имена
    файлов из заглавных букв строчными буквами").
[-] В программе просмотра и редакторе были перепутаны подписи для
    сочетания Alt-F7 в полосе функциональных клавиш.
[-] При включенном параметре "UseNumPad" неправильно работала комбинация
    Shift+Ins (на цифровой клавиатуре).
[+] В параметре командной строки /p можно указывать несколько путей,
    разделенных символом ';'
[+] В параметре "Путь к персональным плагинам" ("Path for personal plugins")
    можно указывать несколько путей, разделенных символом ';'
[!] Изменился порядок загрузки плагинов и приоритеты ключей:
    - если указан пустой /p, то FAR запустится без плагинов
    - если указан /p<path>, то загрузятся только плагины из <path>
    - если указан только ключ /co и кэш плагинов не пуст, то загружаются
      плагины из кэша
    - ключ /co игнорируется, если также указан /p
    - если не указаны ключи /p и /co, то загрузятся плагины из основного
      каталога и из каталога, указанного в параметре "Путь к персональным
      плагинам" ("Path for personal plugins")
[+] В диалог "Системные параметры" ("System settings") добавлена опция
    "Сканировать символические связи" ("Scan symbolic links"). Позволяет
    обрабатывать символические связи наравне с обычными подкаталогами
    при построении дерева каталогов, определении суммарного размера
    файлов в подкаталогах. По умолчанию включена.
[-] Попытка вывода информации об оригинальной папке для символической связи
    в сетевом каталоге (например, в диалоге смены файловых атрибутов)
    приводила к показу ложных данных.
[+] Добавлен режим "числовой" сортировки имен файлов в стиле Windows XP.
    (доступно в меню сортировки Ctrl-F12)
[-] При редактировании описаний для группы файлов обновление панелей
    происходило после ввода последнего описания.
[-] Многократное изменение размеров консоли (Alt-F9) в Windows-9x/Me
    приводили к зависанию системы.
[+] Имена файлов/папок (содержащие символы, перечисленные в правиле
    TechInfo#34) при вставке в редактор/командную строку заключатся в
    кавычки. Новое правило TechInfo#50 управляет этим поведением.
[+] В режиме копирование текста с экрана (Alt-Ins) комбинация Ctrl-U
    снимает выделение.
[-] В меню "Раскраска файлов" ("Files highlighting") комбинация клавиш
    Ctrl-Up перемещала первый элемент на вторую позицию.
[-] Падение FAR Manager при изменении размера консоли внешними командами.
[-] В "Ссылках на папки" ("Folder Shortcuts") выдавалось ошибочное
    сообщение об ошибке при использовании переменных среды в качестве
    пути.
[*] Вставка содержимого буфера обмена (клавишами Ctrl-V или Shift-Ins)
    в окно быстрого поиска производится до тех пор, пока есть совпадение.
[+] Быстрый поиск в панелях. В процессе набора имени можно применять
    функцию транслитерации при ошибочной раскладке клавиатуры.
    В этом случае FAR перекодирует набранный текст и пытается найти
    первое совпадение нового имени.
[*] В TechInfo#10 добавлены "AltFastFindKey" и "FastFindKey" -
    транслитерация в быстром поиске в панелях.
[+] Новая опция "Клик мыши вне диалога закрывает диалог"
    Щелчок левой или правой клавишей мыши за пределами диалога
    закрывает диалог. Эта опция позволяет отключать указанную
    функциональность.
[+] Новое правило TechInfo#52, позволяющее задавать размеры истории
    в позициях для строк ввода в диалогах, посещаемых папок,
    введенных команд, просмотренных и отредактированных файлов.
[+] Правила TechInfo#53 и TechInfo#54 для включения дополнительных
    возможностей FAR по запуску программ.
[+] Новая опция в настройках панелей "Автообновление сетевых дисков".
    Опция включает механизм автоматического обновления панели при
    изменении состояния файловой системы на сетевых дисках.
    По умолчанию - включена.
    Отключение этой опции полезно на медленных сетевых подключениях.
[+] Включение/выключение показа скрытых и системных файлов (Ctrl-H) теперь
    работает при погашенных панелях.
[-] В панелях не выделялась вертикальная разделительная черта, если при
    указании режима панели поле имени стояло не в первой позиции, например,
    при указании режима "S,N" курсор выглядел разорванным на две части.
[!] Введен новый способ разбивать колонки панели на глобальные. Теперь колонки
    разбиваются по наибольшей повторяющейся последовательности. Это позволяет
    использовать в одной глобальной колонке несколько колонок Name и создавать
    колонки вида "S,N" (раньше признаком начала новой колонки было имя файла).
[+] Вместо безликого "CD-ROM" в меню выбора дисков и информационной панели
    теперь выводится информация о типе CD-привода. Различаются CD-ROM,
    CD-RW, DVD-ROM и DVD-RW. Используется функционал, требующий прав
    администратора, в противном случае FAR покажет просто "CD-ROM".
    Работает под Windows 2000 и выше.
[+] Дерево папок. Gray+ и Gray- используются для быстрого перемещения
    вверх или вниз по папкам одного уровня.
[+] Ctrl-Shift-Enter в "Истории папок" (Alt-F12) позволяет перейти в указанную
    папку на пассивной панели
[+] В меню "Файлы" добавлена команда "Уничтожение". Alt-Del в терминальном
    сеансе перехватывается самим терминалом.
[+] Добавлено новое правило (TechInfo#58) - Система: автоматически закрывать
    зависшие программы.
[-] Сочетание Gray-* в диалогах действует только для трехпозиционных
    переключателей (CheckBox).
[*] В комбинированных списках в диалогах клавиша Tab выбирает текущий элемент
    списка (аналогично действию в списке истории поля ввода).
[+] В быстром поиске файлов на панели и диалоге поиска папок добавлена
    возможность перемещаться в обратном направлении по файловой маске с
    помощью Ctrl-Shift-Enter.
[+] После старта FAR Manager выставляет переменную среды %FARHOME%, в которой
    содержится путь к каталогу, откуда FAR был запущен.
[-] Функция транслитерации не учитывала последний символ в перекодировочных
    таблицах.
[-] При удалении записи из истории в полях ввода в диалогах (Shift-Del),
    курсор устанавливался в первую позицию списка.
[+] Новый флаг для колонки размера - SF, размер показывается в виде дробного
    числа, как в Exporer.
[+] Новый флаг для колонки размера - SE, убирает пробел между размером и
    суффиксом.
[+] Новое правило (TechInfo#60) - позволяет игнорировать ошибку
    ERROR_BAD_PATHNAME под Windows 9x/Me, ошибочно возвращаемую операционной
    системой (FindNextFile() из Windows API) в конце получения списка файлов
    пустой директории на сетевом монтированном диске, предоставленном
    сервером, использующим ОС Unix с некоторыми новыми версиями пакета Samba
    (например, v3.0.4).
[+] Новое правило (TechInfo#61) позволяет задать максимальную высоту
    открываемого списка истории в диалогах.
[+] Новое правило (TechInfo#62) позволяющее отключить принудительный вызов
    плагина "Print Manager", назначенный на клавиши Alt-F5.
[+] Новое правило (TechInfo#63) позволяющее задавать максимальный размер
    читаемых данных из файла в который попытались войти из панелей (Enter или
    Ctrl-PgDn).
[+] Новое правило (TechInfo#64) позволяющее контролировать время обновления
    сообщения об удалении файлов.
[-] Диалоги некорректно центрировались на экране.
[+] В диалогах назначения горячей клавиши (меню выбора и конфигурирования
    плагинов) добавлена информация об имени плагина.
[-] Курсор на скрытой панели. Курсор на левой панели, обе отображены. Исполняем
    Ctrl-F2 Ctrl-O Ctrl-F2 Ctrl-F2 Ctrl-O. Курсор на правой панели, а отображена
    левая.
[!] Значение реестра "Panel\AutoChangeFolder" перемещено в
    "Panel\Tree\AutoChangeFolder". Это значит, что опцию "Auto change folder"
    необходимо выставить заново (TechInfo#66)
[*] В правиле TechInfo #33 добавлено значение смещения для прокрутки в
    подсказке.
[!] В команде 'cd' теперь можно использовать символы маски. Например, команда
    'cd progra*' изменяет текущий каталог на Program Files. Т.е. происходит
    переход в первый совпадающий с маской каталог.
[+] Добавлен диалог подтверждения размонтирования SUBST-дисков из меню
    выбора дисков.
[*] Диалоги настройки редактора и программы просмотра переделаны под общий
    стиль диалоговых окон.
[-] Неправильно выводились слишком длинные разделительные линии в диалогах.
[+] В историях просмотра, редактирования и смены папок (Alt-F11 и Alt-F12)
    добавлена комбинация Ctrl-R - обновить список, удалив недоступные
    файлы или каталоги.
[+] Новое правило (TechInfo#68), позволяющее различать левые правые комбинации
    Ctrl-Alt-Shift.
[+] Добавлена опция командной строки "/rc" - после завершения работы
    позволяет восстановить значения заголовка и размеров консольного
    окна, выставленные до запуска FAR Manager.
[-] Параметры можно было сохранить с помощью Shift-F9, при выставленном
    бите 17 в полиси (TechInfo#4)
[+] В 5-м режиме панели в колонке атрибутов символ 'I' означает, что
    файл НЕ индексируется по содержимому..
[+] В 5-м режиме панели в колонке атрибутов символ '$' означает, что
    это т.н. разреженный файл (sparse file).
[+] В 5-м режиме панели в колонке атрибутов символ 'T' означает, что
    это временный файл.

Поиск файлов
~~~~~~~~~~~~
[+] В диалог поиска файлов добавлена опция "Искать в символических связях"
    ("Search in symbolic links"). Позволяет искать файлы в символических
    связях наравне с обычными подкаталогами. По умолчанию (при запуске FAR)
    ее значение равно значению опции "Сканировать символические связи"
    ("Scan symbolic links") из диалога "Системные параметры"
    ("System settings").
[+] Добавлено запоминание таблицы символов в поиске файлов.
[+] В поиске файлов добавлена возможность поиска текста в стандартной
    таблице "Windows текст", наряду с "DOS текст" и "Unicode".
[+] В поиск файлов добавлена возможность поиска по 16-ричным кодам
[!] Размер в результатах поиска выводится с учётом 64-битной математики.
[+] В результатах поиска выводятся атрибуты найденных папок и файлов.
[+] Новая опция "Использовать фильтр" ("Use filter"), включающий возможность
    фильтрования операции поиска по различным параметрам.
[+] Кнопка "Фильтр "("Filter") позволяет настраивать фильтр операций сразу
    в поиске.
[!] При включенном фильтре операций тип даты и времени найденных файлов
    выводится в зависимости от установленных в фильтре:
    Дата модификации, дата создания или дата последнего доступа.
[+] Если используется фильтр операций, то информируем об этом во время поиска.
[-] Неверное позиционирование в архиве после просмотра/редактирования файла
    во время поиска.
[+] Введённая строка текста может разыскиваться не только во всём файле,
    а также в заданном начальном диапазоне, определяемом свойством
    "Проводить поиск в первых" ("Search only in the first"). В случае если
    задано значение, которое меньше размера файла, то остаток файла будет
    игнорироваться, даже если искомая последовательность будет там
    находиться. Настройки диапазона поиска выводятся при нажатии кнопки
    "Дополнительно".
[-] Исправлены проблемы с прорисовкой диалога результатов поиска на
    многопроцессорных компьютерах.
[+] Добавлена опция, позволяющая вести поиск в папках, перечисленных в
    переменной окружения %PATH% (без вложенных папок).

Копирование/Переименование
~~~~~~~~~~~~~~~~~~~~~~~~~~
[*] Прогресс процесса копирования теперь обновляется 4 раза в секунду.
[-] Отключена возможность создания символической связи с локального
    диска на сетевой.
[-] FAR некорректно копировал/перемещал папки с символическими связями.
    Для этого в диалог добавлена опция "[ ] Копировать содержимое
    символических связей". Позволяет уточнять логику работы FAR с
    символическими связями при копировании/перемещении.
    Подробнее см. в помощи.
[-] Ошибки при копировании/перемещении, если среди нескольких целей
    присутствовали "con" или "nul".
[-] FAR выдавал сообщение об ошибке при переименовании каталога, содержимое
    которого отображалось на пассивной панели.
[-] При переименовании файла (Shift-F6) на жестком диске (если на пассивной
    панели компакт-диск и он вынут) FAR показывал меню выбора дисков, не
    закончив процесс переименования.
[-] Перемещение папки из корня в ".." удаляло папку.
[-] Дублирование записей в файле описаний (descript.ion) во время копирования
    нескольких файлов в один (выделяем все, F5, имя файла, Enter, append,
    append........append).
    Теперь в descript.ion помещается описание от последнего файла.
[+] Добавлен фильтр операций в диалоге копирования и перемещения файлов.
[!] Опция "Копировать права доступа" переделана. Дополнительно смотрите
    TechInfo#65 и TechInfo#69.
[-] При создании ссылки на каталог удалялся одноименный файл (если имя ссылки
    совпадало с существующим файлом)
[-] Для копируемых каталогов не выставлялись файловые атрибуты.
[+] В диалоге копирования/перемещение показывается прогресс операции в
    процентах.
[+] Добавлен функционал, позволяющий копировать шифрованные файлы
    за пределы текущего диска. При этом выдается сообщение о
    том, что конечные файлы будут расшифрованы.
    В Windows 2000 для копирования шифрованного файла за пределы
    исходного диска будет использован внутренний механизм копирования,
    независимо от того, включен ли параметр "Использовать системную
    функцию копирования" или нет.

Файловые атрибуты
~~~~~~~~~~~~~~~~~
[-] Ошибка при смене атрибутов с Encrypted на Compressed у шифрованного
    файла/каталога с выставленным атрибутом ReadOnly.
[+] В диалоге установки атрибутов добавлена возможность управления
    атрибутом индексирования файла или папки соответствующим сервисом.
    В 5-м режиме панели в колонке атрибутов символ 'I' означает, что
    файл будет индексироваться.
[+] Диалог установки атрибутов теперь вызывается на плагиновых панелях
    в режиме ReadOnly.

Файловые ассоциации
~~~~~~~~~~~~~~~~~~~
[-] Некорректно формировался файл-список с полными именами (метасимвол "!@F!")
    на панели плагина TempPanel. Каждый файл дополнялся путем из родительской
    панели.
[+] Добавлена возможность в метакоманде "!?<title>?<init>!" задавать
    имя истории. В этом случае команда имеет следующий формат:
    "!?$<history>$<title>?<init>!"

Файловые ассоциации
~~~~~~~~~~~~~~~~~~~
[+] Специальные символы "!&" (список помеченных файлов) и "!&~" (список
    помеченных файлов с короткими именами). В отличие от "!@!" и "!$!"
    подставляются имена файлов, а не имя файла со списком. Не желательно
    использовать "!&" или "!&~" совместно с "IF [NOT] EXIST" в качестве
    параметра проверки "filename" - получите совершенно непредсказуемый
    эффект
[+] Модификатор "!^" обозначающий  ссылку  на  активную  панель.  Например,
    !^!.! обозначает имя текущего файла на активной панели,  а  конструкция
    !#!\!^!.! - файл на  пассивной  панели  с  таким  же  именем,  как  имя
    текущего файла на активной панели.
[+] Для ассоциаций !@! и !$! допускаются следующие модификаторы:
    'Q' - заключать имена с пробелами в кавычки;
    'S' - использовать '/' вместо '\' в путях файлов;
    'F' - использовать полный путь;
    'A' - использовать ANSI кодировку.
    Например,  ассоциация  !@AFQ!  -  имя   файла,   содержащего   имена
    помеченных файлов с  полным  путем  в  ANSI  кодировке;  имена  файлов,
    содержащие пробелы будут заключены в кавычки.

    косая черта '\', например, "..\foo", то:
     1. Копируем только файлы - все файлы будут скопированы в файл "..\foo"

        "..\foo" с сохранением структуры папок.
     3. Копируем файлы и папки - все файлы и папки будут скопированы
        в папку "..\foo" с сохранением структуры папок.
